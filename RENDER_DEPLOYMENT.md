# Royal Shield Unified Backend - Render Deployment

Este checkout despliega el backend Python/FastAPI ubicado en
`royal_shield_backend_unified`. No uses la guia vieja de Node para este deploy.

## Causa probable del fallo actual

El dominio publico `https://server-beckend.onrender.com/` sigue respondiendo
`Servidor activo`, que corresponde al backend Node anterior. El `render.yaml`
actual apunta a FastAPI, pero Render no puede convertir el runtime de un
servicio existente de Node a Python. Si el servicio `server-beckend` ya existe
como Node, debes crear un servicio nuevo o borrar/recrear ese servicio.

## Opcion recomendada: servicio FastAPI nuevo

1. En Render, crea o sincroniza el Blueprint desde este repo.
2. Usa la rama:

```text
royalshield-v1.0.2-unified
```

3. El Blueprint debe crear el servicio:

```text
royal-shield-unified-api
```

4. Render debe ejecutar:

```bash
pip install -r requirements-render.txt
python -m uvicorn api.main:app --host 0.0.0.0 --port $PORT
```

5. El health check debe ser:

```text
/health
```

## Variables requeridas

Render debe tener estas variables como minimo:

```text
ENVIRONMENT=production
LOG_LEVEL=INFO
LOG_FILE=logs/royal_shield_backend.log
DATABASE_URL=<Render Postgres connectionString>
API_INTERNAL_SECRET=<generated by Render>
JWT_SECRET=<generated by Render>
```

No uses `localhost` para `DATABASE_URL` en produccion. Si `DATABASE_URL` falta,
el backend debe fallar rapido para evitar arrancar contra una base local falsa.

## Postgres en Render

El Blueprint incluye una base:

```text
royal-shield-postgres
```

Y conecta `DATABASE_URL` usando:

```yaml
fromDatabase:
  name: royal-shield-postgres
  property: connectionString
```

## Verificacion despues del deploy

Cuando Render termine, prueba:

```bash
curl https://royal-shield-unified-api.onrender.com/health
curl https://royal-shield-unified-api.onrender.com/api/v1/info
```

Respuesta esperada para `/health`:

```json
{
  "status": "healthy",
  "environment": "production"
}
```

## Si necesitas conservar la URL vieja

Si la app Android debe seguir usando:

```text
https://server-beckend.onrender.com
```

entonces en Render debes borrar el servicio Node viejo `server-beckend` y
recrearlo como Python/FastAPI desde este Blueprint. No intentes cambiar el
runtime del servicio existente de Node a Python.

Despues de recrearlo, confirma:

```bash
curl https://server-beckend.onrender.com/health
```

Debe responder JSON con `status: healthy`, no `Servidor activo`.

## Si Render sigue fallando

Revisa los logs de arranque y confirma estos puntos:

- `Environment: production`
- `Initializing database at host:` no debe ser `localhost`
- `/health` debe responder `200`
- La rama desplegada debe ser `royalshield-v1.0.2-unified`
- El servicio debe usar runtime `Python`, no `Node`
