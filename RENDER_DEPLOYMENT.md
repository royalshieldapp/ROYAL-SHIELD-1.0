# Royal Shield Backend - Render Deployment

Este checkout despliega el servicio publico que la app Android usa en:

```text
https://server-beckend.onrender.com
```

## Configuracion correcta

Render debe ejecutar el backend Node ubicado en `backend/`.

```yaml
runtime: node
rootDir: backend
buildCommand: npm ci --omit=dev
startCommand: npm start
healthCheckPath: /health
```

Esto evita el problema de intentar convertir el servicio existente de Node a
Python. El backend Python/FastAPI en `royal_shield_backend_unified/` queda como
referencia/unificado, pero no debe ser el servicio activo de Render mientras la
app Android siga llamando `server-beckend.onrender.com`.

## Variables requeridas

Minimo:

```text
NODE_ENV=production
API_INTERNAL_SECRET=<generated by Render>
JWT_SECRET=<generated by Render>
```

Opcionales segun modulo:

```text
VIRUSTOTAL_API_KEY=
TWILIO_ACCOUNT_SID=
TWILIO_AUTH_TOKEN=
TWILIO_PHONE_NUMBER=
GEMINI_API_KEY=
DID_API_KEY=
VPN_PROVIDER=
VPN_SERVERS=
VPN_SERVER_PUBLIC_KEY=
```

No pongas secretos reales en el repo. Configuralos solo en Render.

## Endpoints que deben responder

Despues del deploy, valida:

```bash
curl https://server-beckend.onrender.com/health
curl https://server-beckend.onrender.com/api/loyalty/status
curl https://server-beckend.onrender.com/api/vpn/servers
curl "https://server-beckend.onrender.com/api/v1/risk-map?bbox_min_lat=25.70&bbox_min_lng=-80.25&bbox_max_lat=25.82&bbox_max_lng=-80.15"
```

`/health` debe responder `status: healthy`.

## Si Render sigue en timeout

Revisa en Render:

- El servicio activo debe llamarse `server-beckend`.
- Branch: `royalshield-v1.0.2-unified`.
- Root Directory: `backend`.
- Runtime: `Node`.
- Build Command: `npm ci --omit=dev`.
- Start Command: `npm start`.
- Health Check Path: `/health`.
- Ejecuta un redeploy con clear build cache si sigue usando configuracion vieja.
